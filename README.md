# 🎬 영화 리뷰 플랫폼 포인트 생태계 구축

영화 리뷰 서비스는 한 번 들어와서 글을 읽고 나가는 구조로 끝나기 쉽다고 생각했습니다.  
그래서 팀 프로젝트에서는 사용자의 행동이 자연스럽게 다음 행동으로 이어지도록, **포인트를 중심으로 한 참여 구조**를 설계하고 구현했습니다.

출석하고, 퀘스트를 수행하고, 퀴즈를 풀고, 룰렛을 돌려 포인트를 모은 뒤  
상점에서 아이템을 구매하거나 선물하고, 인벤토리에서 실제로 적용하는 흐름까지 연결했습니다.

> 참여 → 보상 → 소비 → 프로필 변화 → 재방문

---

## 프로젝트 소개

- **프로젝트명**: 영화 리뷰 플랫폼
- **담당 영역**: 포인트 생태계 백엔드 전담
- **구현 목표**: 단순 조회형 서비스가 아니라, 사용자가 다시 들어올 이유가 생기는 구조 만들기

제가 맡은 범위는 아래와 같습니다.

- 출석체크와 연속 출석 보상
- 일일 퀘스트 / 데일리 퀴즈 / 룰렛 보상 구조
- 포인트 적립·차감 로직 통합
- 포인트 상점, 구매, 선물, 환불
- 위시리스트, 인벤토리, 아이템 사용·장착
- 포인트 이력 관리
- 관리자용 포인트 / 상품 / 자산 운영 기능

---

## 핵심 구현 내용

### 1) 포인트를 하나의 공통 재화로 통합
리뷰 서비스 안에서 흩어질 수 있는 여러 기능을 포인트 하나로 연결했습니다.  
사용자는 활동으로 포인트를 모으고, 모은 포인트로 아이템을 소비하며, 소비 결과가 다시 프로필 변화로 이어지도록 구성했습니다.

### 2) 적립과 차감의 정합성 보장
포인트는 단순 숫자가 아니라 서비스 안에서 실제로 순환하는 재화라고 생각했습니다.  
그래서 포인트 증감은 한 곳에서만 처리하도록 묶었고, 차감 시 잔액을 검증했으며, 변경 내역은 반드시 이력 테이블에 남기도록 구현했습니다.

### 3) 사용자 경험과 운영 기능까지 함께 고려
사용자 기능만 만드는 데서 끝내지 않고, 운영자가 포인트와 아이템을 직접 관리할 수 있는 관리자 기능도 함께 정리했습니다.  
덕분에 실제 서비스 관점에서 운영 가능한 구조까지 고려할 수 있었습니다.

## 구현 기능 요약

### 사용자 기능
- 출석체크, 출석 달력, 연속 출석 반영
- 일일 퀘스트 조회 / 진행도 누적 / 보상 수령
- 데일리 퀴즈 랜덤 출제 / 정답 검증
- 룰렛 티켓 사용 / 랜덤 보상 지급
- 상점 목록 조회 / 검색 / 페이징
- 상품 구매 / 선물 / 환불
- 위시리스트 등록·해제 / 내 찜 목록 조회
- 인벤토리 조회 / 아이템 사용 / 장착 / 해제
- 포인트 이력 조회

### 관리자 기능
- 회원 포인트 지급 / 회수
- 데일리 퀴즈 등록 / 수정 / 삭제
- 상점 아이템 등록 / 수정 / 삭제
- 유저 인벤토리 / 아이콘 자산 조회 및 지급·회수

## 기술 스택 및 협업 도구

- ⚙️ **Backend**: Spring Boot
- 📦 **Data Access**: MyBatis
- 🗄️ **Database**: Oracle
- 🔐 **Authentication**: JWT
- 🧱 **Architecture**: MVC Pattern
- 🤝 **Collaboration**: Git, GitHub

요청은 `Controller → Service → DAO → Database` 흐름으로 처리되며,  
인증이 완료되면 `loginId`를 `RequestAttribute`로 주입해 서비스 로직이 로그인 사용자 기준으로 동작하도록 구성했습니다.
